# 房间可见性系统设计文档

## 问题

墙壁有厚度后，当相机面向房间时，墙壁会遮挡视线，无法看清屋内物品。项目将来会扩展为多个房间 + 窗外风景，需要一个方案让观察者无障碍地看到室内和室外。

## 核心思路

两个独立的问题，分开解决：

1. **墙挡住视线** → 动态隐藏/半透明（RTS/Sims 风格）
2. **多个房间互相干扰** → 房间级可见性（只渲染当前房间）

---

## 方案一：墙壁动态隐藏/半透明

### 原理

检测哪些墙壁位于相机和房间之间，将这些墙壁隐藏或变为半透明。

### 实现方式

- 每帧从相机位置向房间中心做射线检测
- 命中的墙壁 mesh 设置 `visible = false` 或调整 `material.opacity`
- 可以做淡入淡出动画（opacity 从 1 → 0 过渡）

### 优点

- 视觉效果直观，像 The Sims / RTS 游戏那样
- 可以做平滑的淡入淡出过渡动画
- 房间结构保持完整，只是视觉上透明

### 缺点

- 半透明时会看到墙背后的杂物（其他房间的家具等），视觉上比较乱
- 需要设置材质为 transparent，可能影响渲染排序
- 需要判断哪些墙"挡在相机和房间之间"的逻辑

### 适用场景

- 单房间场景（当前项目）
- 相机围绕房间旋转时，自动隐藏面向相机的墙壁

---

## 方案二：房间级可见性

### 原理

每个房间是一个独立的 `THREE.Group`，根据角色所在房间，只渲染当前房间，其他房间直接隐藏。

### 渲染内容

```
渲染内容 = 当前房间 + 户外场景 + 角色
```

- 当前房间：所有墙壁、家具、装饰物
- 户外场景：天空、花园、树木等，始终渲染
- 角色：始终渲染

### 场景结构

```
scene
├── outdoor (天空/花园/风景) — 始终可见
├── roomA.group — 角色在此房时可见
│   ├── walls
│   ├── furniture
│   └── windowGlass (透明，采用户外场景)
├── roomB.group — 角色不在时 hidden
│   ├── walls
│   ├── furniture
│   └── windowGlass
└── character — 始终可见
```

### 窗户和门的处理

- **窗户**：玻璃材质透明，背后是户外场景（天空盒或 cubemap），透过窗户可以看到外面
- **门**：门洞是开口，门外渲染户外或走廊场景
- **墙壁**：不透明，天然遮挡其他房间（因为其他房间根本不渲染）

### 房间切换

角色从房间 A 走到房间 B 时：
- **硬切**：瞬间切换，最简单
- **淡入淡出**：旧房间淡出 → 新房间淡入，体验更好
- **走廊过渡**：走廊作为独立区域，进入走廊时两个房间都不显示

### 优点

- 逻辑简单，不需要复杂的遮挡判断
- 性能好，只渲染一间房的物体
- 天然支持多房间扩展
- 墙壁不透明，不会看到其他房间的杂物

### 缺点

- 门打开时无法看到隔壁房间（除非额外实现 render-to-texture）
- 需要管理房间分组和切换逻辑

### 适用场景

- 多房间项目（未来规划）
- 需要窗外风景的场景

---

## 推荐策略

### 当前阶段（单房间）

使用**方案一（动态隐藏/半透明）**：
- 相机围绕房间旋转时，自动隐藏面向相机的墙壁
- 做淡入淡出动画，体验像 RTS 游戏

### 未来阶段（多房间 + 户外）

结合**方案一 + 方案二**：
- 房间级可见性作为基础（只渲染当前房间）
- 当前房间内部再用动态隐藏/半透明处理相机遮挡
- 户外场景始终渲染，窗户作为"窗口"

---

## 实现要点

### 方案一的关键点

1. 每帧检测相机和房间中心之间被哪些墙遮挡
2. 被遮挡的墙设置 `material.transparent = true`，`material.opacity` 做动画
3. 需要处理墙壁的多个 mesh piece（后墙有4块、前墙有3块）
4. 淡入淡出用 lerp 实现，避免突兀切换

### 方案二的关键点

1. 每个房间封装为独立的 `THREE.Group`
2. 检测角色位置，判断当前所在房间
3. 切换时设置其他房间 `.visible = false`
4. 户外场景独立于房间，始终渲染
5. 窗户玻璃用透明材质，采用户外场景作为背景
